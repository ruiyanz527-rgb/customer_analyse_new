     • 字段数: 15个维度
     • 包含: 结构化特征 + 非结构化文本 + 推理链路 + 奖励信号

2. 📝 SFT训练数据
   - kyc_sft_training_data.jsonl (7.6 MB)
     • 格式: JSONL (每行一条JSON)
     • 样本数: 10,000 条
     • 包含: prompt → target 对应关系

3. 📚 课程学习数据
   - kyc_curriculum_stage_1_data.csv (1.1 MB, 2,800 条)
     特点: 明显低风险或高风险样本 (简单)
   - kyc_curriculum_stage_2_data.csv (3.5 MB, 5,600 条)
     特点: 1:1混合简单和困难样本

4. ⚙️ 超参配置文件
   - kyc_grpo_config.json (开箱即用)
     算法: GRPO
     模型: GLM-4-9B-Chat
     学习率: 1e-5
     重要超参:
       • actor_entropy_coeff: 0.001 (避免熵坍缩)
       • actor_clip_ratio_high: 0.28 (DAPO策略)
       • kl_loss_coeff: 0.003 (KL正则)
       • kl_loss_type: k2 (GRPO推荐)

📊 数据集特征分析
================================================================================

【用户基础特征】
  ✓ 年龄范围: 20-65 岁
  ✓ 职业分类: 9 种
  ✓ 职位库: 45+ 种具体职位
  ✓ 城市覆盖: 10 个主要城市
  ✓ 学历等级: 5 级
  ✓ 工作年限: 0-45 年
  ✓ 收入范围: ¥3,000 - ¥35,000+
  ✓ 芝麻分范围: 300-850

【非结构化文本】
  ✓ kyc_raw_text: 模拟真实KYC申请文本
    - 包含用户基本信息、职业描述、收入水平
    - 三种模板变体，增加自然性
    - 平均长度: 150-200 字符

【推理链路标注】
  ✓ reasoning_chain: 4步人工审核推理过程
    Step 1: [职业风险] 职业类别风险评估
    Step 2: [收入警告] 收入合理性检查
    Step 3: [交易行为] 交易特征分析
    Step 4: [综合评分] 最终风险结论

【奖励信号】
  ✓ risk_score: 连续型奖励 [0.0-1.0]
    - 基于职业基础风险
    - 考虑年龄、收入、芝麻分等因素
  ✓ is_risky_user: 二分类标签 [0, 1]
    - 高风险占比: 15-20%
